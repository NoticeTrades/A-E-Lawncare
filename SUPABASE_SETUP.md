# Supabase Setup (A&E Lawn Care)

## 1) Create project
- Go to [https://supabase.com](https://supabase.com) and create a new project.
- In Project Settings -> API, copy:
  - Project URL
  - `anon` public key

## 2) Add keys to site
- Open `supabase-config.js`
- Set:
  - `window.AE_SUPABASE_URL`
  - `window.AE_SUPABASE_ANON_KEY`
  - `window.AE_ADMIN_EMAIL` (optional label only)

## 3) Create `quotes` table
Run this SQL in Supabase SQL Editor:

```sql
create table if not exists public.quotes (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  name text not null,
  email text not null,
  phone text not null,
  street_address text,
  city text,
  state text,
  zip_code text,
  customer_type text,
  service text not null,
  message text not null
);
```

### Add quote form columns (extended form)

If the table already exists from an earlier version, run this once to add address and customer fields:

```sql
alter table public.quotes add column if not exists street_address text;
alter table public.quotes add column if not exists city text;
alter table public.quotes add column if not exists state text;
alter table public.quotes add column if not exists zip_code text;
alter table public.quotes add column if not exists customer_type text;
-- Email is required in the app now; make sure column allows null or set not null after backfill:
alter table public.quotes alter column email set default '';
```

Optional: require email for new rows (only after you are sure legacy rows are fine):

```sql
alter table public.quotes alter column email set not null;
```

## 4) Enable Row Level Security
Run:

```sql
alter table public.quotes enable row level security;
```

## 5) Policies
- Public users can submit quote requests
- Only signed-in owners can read/delete

Run:

```sql
create policy "Public can insert quotes"
on public.quotes
for insert
to anon
with check (true);

create policy "Authenticated can read quotes"
on public.quotes
for select
to authenticated
using (true);

create policy "Authenticated can delete quotes"
on public.quotes
for delete
to authenticated
using (true);
```

## 6) Create owner login
- Supabase Dashboard -> Authentication -> Users -> Add user
- Create owner email/password account.
- Use that email/password on `admin.html`.

## 7) Access points
- Contact form submits to Supabase:
  - `contact.html`
- Owner dashboard login + submissions:
  - `admin.html`

## Notes
- Keep the `anon` key in front-end (this is normal).
- Never put Supabase `service_role` key in website files.
